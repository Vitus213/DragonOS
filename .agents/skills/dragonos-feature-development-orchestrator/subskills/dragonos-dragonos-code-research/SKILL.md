# DragonOS 现状调研

## 目标

在不修改代码的前提下，快速建立目标功能在 DragonOS 中的实现全景，输出可用于后续设计和评审的证据化结论。

## 输入

- 目标功能或子系统（例如 cgroup、epoll、futex）
- 相关失败现象（可选）
- 优先关注维度（语义/性能/并发/安全）

## 工作步骤

1. 用 `Glob` 定位候选目录（优先 `kernel/`、`docs/`、`user/apps/c_unitest/`）。
2. 用 `Grep` 检索入口与调用链（syscall、trait impl、核心结构体、错误码）。
3. 用 `Read` 抽取最小必要上下文，记录 `file:line` 证据。
4. 归纳当前行为、缺口与风险，不给 workaround 方案。

## 输出格式

```markdown
## DragonOS 现状

### 1) 调用链
- 用户态入口 -> 内核入口 -> 子系统路径（附 file:line）

### 2) 关键结构
| 结构/接口 | 作用 | 关键字段/方法 | 证据 |
|---|---|---|---|

### 3) 当前语义与限制
| 场景 | 当前行为 | 与 Asterinas 参考语义差异 | 风险等级 | 证据 |
|---|---|---|---|---|

### 4) 需要后续确认的问题
- ...
```

## 质量要求

- 所有结论必须附 `file:line`。
- 区分“已实现”“部分实现”“未实现”。
- 明确并发与资源释放路径是否完整。
